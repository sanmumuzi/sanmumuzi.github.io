Learn Docker

docker学习笔记

基本概念

镜像（Image）

容器（Container）

镜像（Image）和容器（Container）的关系，就像是面向对象程序设计中的类和实例一样，镜像是静态的定义，容器是镜像运行时的实体。

每一个容器运行时，是以镜像为基础层，在其上创建一个当前容器的存储层，我们可以称这个为容器运行时读写而准备的存储层为容器存储层。

容器消亡时，容区存储层也随之消亡。

所有的文件写入操作，都应该使用数据卷（Volume）

仓库（Repository）

安装Docker(此下均为ubuntu16.04安装)

阿里云的安装脚本

curl -sSL http://acs-public-mirror.oss-cn-hangzhou.aliyuncs.com/ docker-engine/internet | sh

添加APT镜像源

$ sudo apt-get update

$ sudo apt-get install apt-transport-https ca-certificate

为了确认所下载软件包的合法性，需要添加 Docker 官方软件源的 GPG 密钥

$ sudo apt-key adv –keyserver hkp://p80.pool.sks-keyservers.net :80 –recv-keys 58118E89F3A912897C070ADBF76221572C52609D

向source.list 中添加Docker软件源

Xenial 16.04 (LTS)
deb https://apt.dockerproject.org/repo ubuntuxenial main
为下面的REPO

$ echo “REPO” | sudo tee /etc/apt/sources.list.d/docker.list

添加成功后，更新apt软件包缓存

$ sudo apt-get update

安装Docker

$ sudo apt-get install docker-engine

启动Docker引擎

$ sudo systemctl enable docker $ sudo systemctl start docker

建立Docker用户组

$ sudo groupadd docker

$ sudo usermod -aG docker $username

使用

docker ps (-a)

查看正在运行(所有)docker容器

docker rm 容器名

删除某个容器

docker rmi 镜像名

删除某个镜像

docker pull {选项} 地址 仓库名:标签

获取镜像

eg: docker pull ubuntu:14.04
(默认地址是Docker Hub, 默认为官方镜像)

docker run -it –rm ubuntu:14.04 bash

以镜像为基础启动一个容器来运行

启动其中的bash并进行交互式操作

-it -i:交互式操作，-t:终端

–rm 容器退出后随之将其删除

ubuntu:14.04 用该镜像为基础启动容器

bash 交互式终端

docker images (列出镜像)

docker images -f dangling=true (显示虚悬镜像（dangling image）)

docker rmi $(docker images -q -f dangling=true) (删除虚悬镜像)

docker images -a （显示所有镜像（包括中间层镜像））

docker images ubuntu (列出部分镜像)

docker iamges -q (只显示ID号)

docker run –name webserver -d -p 80:80 nginx

使用nginx镜像启动一个容器，命名为webserver，映射80端口(前面的数字)。

当使用docker run 来创建容器时，Docker 在后台进行的标准操作包括：

  • 检查本地是否存在指定的镜像，不存在就从公有仓库下载
  • 利用镜像创建并启动一个容器
  • 分配一个文件系统，并在只读的镜像层外面挂载一层可读写层
  • 从宿主主机配置的网桥接口中桥接一个虚拟接口到容器中去
  • 从地址池配置一个ip地址给容器
  • 执行用户指定的应用程序
  • 执行完毕后容器被终止

docker exec -it webserver bash (进入名为webserver的容器)

docker diff webserver (查看具体的改动)

docker commit –author “作者名” –message “提示信息” (容器名) （ 依据镜像）

docker history (容器名)

eg: docker history nginx:v2

docker start (容器名) （启动已终止容器）

docker run -d ubuntu:14.04 （-d参数，后台运行）

docker logs [container ID or NAMES] （获取容器的输出信息）

docker stop [container ID or NAMES] （终止容器）

docker restart [container ID or NAMES] （将一个运行态的容器终止，然后重新启动它）
